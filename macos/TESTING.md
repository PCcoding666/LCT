## 测试概述

LCT 应用的测试分为以下几个层次：

| 测试类型 | 目的 | 运行方式 |
|---------|------|---------|
| 单元测试 | 测试独立的函数和类 | 自动化 (swift test) |
| 集成测试 | 测试组件间的交互 | 手动 + 部分自动化 |
| 功能测试 | 验证用户可见的功能 | 手动测试 |
| 性能测试 | 确保性能达标 | 手动监控 |

## 自动化单元测试

### 方法一：使用 Xcode 运行测试（推荐）

1. 用 Xcode 打开项目：
   ```bash
   cd macos
   open Package.swift
   ```

2. 在 Xcode 中添加测试目标：
   - 选择 `File` > `New` > `Target`
   - 选择 `Unit Testing Bundle`
   - 命名为 `LCTMacTests`

3. 将测试文件添加到测试目标：
   - 拖拽 `Tests/LCTMacTests/*.swift` 到测试目标

4. 使用快捷键 `⌘ + U` 运行所有测试

5. 或者在 Test Navigator (`⌘ + 6`) 中选择特定测试运行

### 方法二：创建独立测试脚本

```bash
# 运行快速测试验证脚本
./Scripts/run_tests.sh
```

### 方法三：手动验证核心逻辑

在 Swift REPL 中测试核心函数：
```bash
swift repl
```

```swift
// 测试 CJK 检测
let text = "Hello 世界"
text.contains { char in
    for scalar in char.unicodeScalars {
        if (0x4E00...0x9FFF).contains(scalar.value) { return true }
    }
    return false
}
// 预期输出: true
```

### 测试覆盖的模块

| 模块 | 测试文件 | 测试内容 |
|------|---------|---------|
| TextUtils | `TextUtilsTests.swift` | CJK 检测、标点处理、文本处理 |
| Models | `ModelsTests.swift` | TranslationEntry、AppSettings、语言枚举 |
| OllamaService | `OllamaServiceTests.swift` | 错误类型、API 结构体 |

### 编写新测试

测试文件位于 `Tests/LCTMacTests/` 目录。遵循以下命名约定：

```swift
func test<功能>_<场景>_<预期结果>() {
    // Given (准备)
    let input = "测试数据"
    
    // When (执行)
    let result = functionToTest(input)
    
    // Then (断言)
    XCTAssertEqual(result, expectedValue)
}
```

## 手动功能测试

### 1. 首次启动测试

**测试目标**：验证首次使用向导正常工作

**步骤**：
1. 删除所有 LCT 数据（模拟首次安装）：
   ```bash
   rm -rf ~/Library/Application\ Support/LCT
   rm ~/Library/Preferences/com.yourcompany.LCT.plist
   ```

2. 启动应用

3. **验证**：
   - [ ] 显示欢迎界面
   - [ ] 权限请求对话框正确弹出
   - [ ] Ollama 检测正常工作
   - [ ] 可以下载/选择 AI 模型
   - [ ] 完成设置后进入主界面

### 2. 权限测试

**测试目标**：验证权限请求和权限缺失处理

**步骤**：
1. 在系统设置中撤销 LCT 的麦克风权限
2. 启动应用并尝试捕获麦克风音频
3. **验证**：
   - [ ] 应用显示权限缺失提示
   - [ ] 提供引导用户授权的说明

重复测试屏幕录制权限和语音识别权限。

### 3. 语音识别测试

**测试目标**：验证语音识别功能

**步骤**：
1. 启动应用
2. 选择源语言（如 English）
3. 点击 "Start" 按钮
4. 播放或说出一段英语

**验证**：
- [ ] 音频电平指示器有反应
- [ ] 识别结果显示在主窗口
- [ ] 识别准确度可接受（>80%）
- [ ] 实时更新识别内容

**多语言测试矩阵**：

| 源语言 | 测试内容 | 预期准确率 |
|--------|---------|-----------|
| en-US | "Hello, how are you today?" | >90% |
| zh-CN | "今天天气怎么样？" | >85% |
| ja-JP | "こんにちは" | >85% |
| ko-KR | "안녕하세요" | >80% |

### 4. 翻译测试

**测试目标**：验证 Ollama 翻译功能

**前置条件**：
- Ollama 服务正在运行
- AI 模型已下载

**步骤**：
1. 确认 Ollama 状态（绿色指示器）
2. 设置目标语言为中文
3. 开始捕获并说英语
4. 等待翻译结果

**验证**：
- [ ] 翻译结果显示在识别文本下方
- [ ] 延迟时间显示正确
- [ ] 翻译质量可接受
- [ ] 上下文感知翻译工作正常（连续句子翻译更准确）

**翻译质量测试**：

| 原文 | 目标语言 | 预期翻译（示例） |
|------|---------|-----------------|
| "Good morning" | 中文 | "早上好" |
| "Thank you very much" | 日语 | "ありがとうございます" |
| "How can I help you?" | 中文 | "我能帮你什么？" |

### 5. 悬浮窗口测试

**测试目标**：验证悬浮窗口功能

**步骤**：
1. 启用悬浮窗口（`⌘ + O` 或设置）
2. 开始实时翻译
3. 测试以下功能：

**验证**：
- [ ] 悬浮窗口始终置顶
- [ ] 可以拖动到任意位置
- [ ] 可以调整大小
- [ ] 显示最近的翻译内容
- [ ] 透明度设置生效
- [ ] 字体大小设置生效
- [ ] 在全屏应用中仍然可见

### 6. 历史记录测试

**测试目标**：验证历史记录功能

**步骤**：
1. 进行一些翻译操作
2. 打开历史窗口（`⌘ + ⇧ + H`）

**验证**：
- [ ] 历史记录正确保存
- [ ] 搜索功能正常工作
- [ ] 可以按日期筛选
- [ ] 可以复制单条记录
- [ ] CSV 导出功能正常
- [ ] 清除历史功能正常

### 7. 设置测试

**测试目标**：验证所有设置选项

**测试设置项**：

| 设置项 | 测试方法 | 预期行为 |
|--------|---------|---------|
| 源语言 | 切换语言后识别 | 识别对应语言 |
| 目标语言 | 切换后翻译 | 翻译到新语言 |
| Ollama 模型 | 切换模型 | 使用新模型翻译 |
| 温度 | 调整后翻译 | 翻译风格变化 |
| 悬浮窗口透明度 | 拖动滑块 | 实时预览透明度变化 |
| 字体大小 | 调整后查看 | 字体大小正确变化 |
| 上下文条目数 | 调整后翻译 | 翻译连贯性变化 |

### 8. 快捷键测试

**测试目标**：验证所有快捷键功能

| 快捷键 | 预期功能 | 测试结果 |
|--------|---------|---------|
| `⌘ + Space` | 开始/停止捕获 | [ ] |
| `⌘ + P` | 暂停/继续 | [ ] |
| `⌘ + ,` | 打开设置 | [ ] |
| `⌘ + ⇧ + C` | 复制翻译 | [ ] |
| `⌘ + O` | 切换悬浮窗口 | [ ] |
| `⌘ + ⇧ + H` | 查看历史 | [ ] |

## 集成测试

### 1. Ollama 集成测试

**测试 Ollama 连接**：

```bash
# 检查 Ollama 是否运行
curl http://localhost:11434/api/tags

# 测试模型响应
curl http://localhost:11434/api/generate -d '{
  "model": "qwen3.5:4b-mlx",
  "prompt": "Hello",
  "stream": false
}'
```

**在应用中测试**：
1. 停止 Ollama 服务，验证应用显示"未连接"
2. 启动 Ollama 服务，验证应用自动重连
3. 切换不同模型，验证翻译正常

### 2. 系统音频捕获测试

**步骤**：
1. 启用系统音频捕获
2. 播放 YouTube 视频或 Zoom 会议
3. 验证音频被正确捕获和识别

**注意**：需要屏幕录制权限

### 3. 麦克风捕获测试

**步骤**：
1. 启用麦克风捕获
2. 对着麦克风说话
3. 验证音频被正确捕获和识别

### 4. 同时捕获测试

**步骤**：
1. 同时启用系统音频和麦克风
2. 播放音频同时说话
3. 验证两个音频源都被正确处理

## 性能测试

### 1. 内存使用监控

**工具**：Activity Monitor 或 Instruments

**测试场景**：
- 空闲状态：< 100 MB
- 正常使用（识别+翻译）：< 500 MB
- 长时间运行（1小时）：无明显内存增长

**检查内存泄漏**：
```bash
# 使用 Instruments 运行泄漏检测
leaks -atExit -- /path/to/LCT
```

### 2. CPU 使用监控

**测试场景**：
- 空闲状态：< 5%
- 正常识别：< 30%
- 翻译处理：短暂峰值可接受

### 3. 翻译延迟测试

**测试目标**：验证翻译延迟在可接受范围内

| 模型 | 期望延迟 |
|------|---------|
| gemma2:2b | < 500ms |
| qwen3.5:4b-mlx | < 1000ms |
| qwen2.5:7b | < 2000ms |

### 4. 长时间运行测试

**步骤**：
1. 启动应用并开始捕获
2. 持续运行 1-2 小时
3. 监控内存、CPU、功能稳定性

**验证**：
- [ ] 无内存泄漏
- [ ] 无崩溃
- [ ] 功能正常

## 常见测试问题

### 测试编译失败

```bash
# 清理并重新构建
swift package clean
swift build
swift test
```

### 权限问题

确保在系统设置中授予 Terminal/Xcode 相应权限进行测试。

### Ollama 相关测试失败

确保 Ollama 服务正在运行：
```bash
ollama serve
```

并且测试所需的模型已下载：
```bash
ollama pull qwen3.5:4b-mlx
```

## 持续集成 (CI)

### GitHub Actions 配置示例

```yaml
name: Tests

on: [push, pull_request]

jobs:
  test:
    runs-on: macos-latest
    steps:
      - uses: actions/checkout@v4
      
      - name: Build
        run: swift build -c release
        working-directory: macos
        
      - name: Run Tests
        run: swift test
        working-directory: macos
```

**注意**：由于 LCT 依赖系统权限（麦克风、屏幕录制），完整的集成测试需要在本地运行。CI 主要运行单元测试。
